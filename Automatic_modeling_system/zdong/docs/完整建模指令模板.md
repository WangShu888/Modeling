# 完整建模指令模板

版本：`v0.1`

日期：`2026-03-30`

适用范围：`基于 CAD/PDF 图纸与文本要求生成 IFC，并由用户使用 Revit 查看结果`

## 1. 文档目标

本文档用于定义建筑自动建模系统中的完整建模指令模板，作为以下三个层面的统一输入标准：

- 用户输入模板
- AI 需求转化模板
- 系统结构化数据模板

模板的目标不是让用户写复杂的技术指令，而是让系统能够稳定地将输入转化为可执行的建模命令。

## 2. 为什么需要完整模板

完整建模指令模板的作用如下：

- 提高用户输入完整度
- 降低 AI 需求理解偏差
- 为规则引擎提供清晰约束来源
- 为参数化建模与 BIM 引擎提供标准输入
- 支持后续版本对比、审计与回放

如果没有统一模板，容易出现以下问题：

- 用户每次描述方式不同
- AI 输出格式不稳定
- 规则无法判断哪些字段缺失
- BIM 引擎无法稳定建模

## 3. 模板分层

建议使用三层模板体系：

### 3.1 用户输入模板

面向用户，表达尽量自然。

### 3.2 AI 转化模板

面向大模型，约束输出必须为固定结构。

### 3.3 系统结构化模板

面向后端模块，作为 `DesignIntent / ModelPatch / ExportConfig` 等对象的标准定义。

## 4. 用户输入模板

### 4.1 推荐输入结构

用户输入建议由“图纸文件 + 文本指令 + 可选表单”组成。

#### 图纸文件

- `CAD/DWG/DXF`
- `PDF`

#### 文本指令

建议覆盖以下信息：

- 建模目标
- 建筑类型
- 图纸说明
- 层数与层高
- 构件修改要求
- 输出要求
- 最终使用软件

### 4.2 用户输入模板示例

```text
请根据我上传的图纸建立 BIM 模型。

项目类型：住宅楼
图纸类型：标准层 CAD 平面图 + 立面 PDF
总层数：18 层
标准层层高：3.0m
首层层高：4.5m
建模要求：根据图纸建立墙、板、门、窗、空间等 BIM 模型
修改要求：将所有 800x1200 的窗替换为落地窗
输出要求：导出 IFC 文件
最终用途：使用 Revit 打开查看结果
```

### 4.3 用户输入最小模板

若做简化版本，至少要求用户填写：

- 上传图纸
- 建筑类型
- 主要修改要求
- 输出格式

示例：

```text
根据上传 CAD 图纸建立住宅 BIM 模型，将所有 800x1200 的窗改为落地窗，输出 IFC，用 Revit 查看。
```

### 4.4 无图纸文本建模模板

当用户不上传图纸时，建议明确标注为“无图纸，文本建模”，避免与图纸驱动模板混淆。

推荐模板如下：

```text
项目类型：住宅楼
输入方式：无图纸，文本建模
总层数：2层
首层层高：4.5m
二层层高：3.0m
首层建筑面积：80㎡
建模要求：建立一个两层住宅楼，室内设置一个主楼梯，建筑外侧设置一个可直达二层的外楼梯；二层设置一个阳台；一层包含厨房和客厅；二层包含3个卧室，且每个卧室带独立卫生间
补充要求：未说明部分由系统按常规住宅参数自动补全
输出要求：导出 IFC 文件
最终用途：使用 Revit 打开查看结果
```

使用说明：

- 两层建筑建议直接区分“首层层高”和“二层层高”，不建议写“标准层层高”
- “首层面积 80㎡”建议明确为建筑面积，而不是套内面积
- 楼梯、阳台等描述尽量写明数量和位置关系
- 若输入不完整，系统可生成方案级模型，但必须写入 `Assumptions`

## 5. AI 需求转化模板

### 5.1 使用目标

将用户输入转换为结构化结果，输出内容必须满足固定 `JSON Schema`。

### 5.2 提示模板示例

```text
你是建筑自动建模系统的需求转化模块。

请根据以下输入内容：
1. 用户文本需求
2. 图纸解析结果
3. 表单字段

输出严格符合 JSON Schema 的结构化结果，不要输出解释性文字。

输出必须包含：
- DesignIntent
- MissingFields
- Assumptions
- CompletionTrace
- ModelPatch
- ElementSelector

请遵循以下规则：
- 用户明确提出的要求优先级最高
- 图纸解析结果高于默认模板
- 缺失但非关键字段可补全
- 所有补全字段必须写入 Assumptions
- 涉及法规和交付的关键字段若缺失，必须写入 MissingFields
- 若用户提出构件修改要求，必须生成 ElementSelector 和 ModelPatch
```

### 5.3 转化策略

- 用户说“根据图纸建模”，转为 `source_mode = cad_to_bim`
- 用户说“生成住宅楼”，转为 `building_type = residential`
- 用户说“把所有 800x1200 的窗替换为落地窗”，转为：
  - `ElementSelector`
  - `ModelPatch`

## 6. 系统结构化模板

### 6.1 DesignIntent 模板

```json
{
  "project_id": "proj_001",
  "source_mode": "cad_to_bim",
  "source_assets": [
    "asset_plan_001",
    "asset_elevation_001"
  ],
  "building_type": "residential",
  "site": {
    "boundary_source": "drawing_or_uploaded_polygon",
    "north_angle": 0
  },
  "constraints": {
    "floors": 18,
    "standard_floor_height_m": 3.0,
    "first_floor_height_m": 4.5,
    "ruleset": "cn_residential_v1"
  },
  "program": {
    "spaces_from_drawings": true
  },
  "style": {
    "facade": "follow_drawing"
  },
  "deliverables": ["ifc"],
  "final_use": "revit_view"
}
```

```json
{
  "project_id": "proj_002",
  "source_mode": "text_only",
  "source_assets": [],
  "building_type": "residential",
  "site": {
    "boundary_source": "unspecified"
  },
  "constraints": {
    "floors": 2,
    "first_floor_height_m": 4.5,
    "second_floor_height_m": 3.0,
    "first_floor_area_sqm": 80,
    "ruleset": "cn_residential_v1"
  },
  "program": {
    "first_floor_spaces": ["kitchen", "living_room"],
    "second_floor_spaces": [
      "bedroom_suite_1",
      "bedroom_suite_2",
      "bedroom_suite_3"
    ],
    "stairs": [
      {"type": "internal_main_stair", "count": 1},
      {"type": "external_stair_to_second_floor", "count": 1}

  },

    {
      "field": "site.boundary_source",

    }
  ],
  "deliverables": ["ifc"],
  "final_use": "revit_view"

```

### 6.2 MissingFields 模板

```json
{
  "missing_fields": [
    "site.boundary",
    "constraints.height_limit_m"
  ]
}
```

### 6.3 Assumptions 模板

```json
{
  "assumptions": [
    {
      "field": "constraints.standard_floor_height_m",
      "value": 3.0,
      "source": "residential_template_default"
    }
  ]
}
```

### 6.4 CompletionTrace 模板

```json
{
  "completion_trace": [
    {
      "field": "constraints.standard_floor_height_m",
      "source_type": "template_default",
      "source_ref": "residential_template_v1",
      "confidence": 0.92
    }
  ]
}
```

### 6.5 ElementSelector 模板

```json
{
  "selector": {
    "ifc_type": "IfcWindow",
    "properties": {
      "overall_width_mm": 800,
      "overall_height_mm": 1200
    }
  }
}
```

### 6.6 ModelPatch 模板

```json
{
  "action": {
    "type": "replace_family",
    "target_family": "floor_to_ceiling_window"
  },
  "scope": {
    "building_id": "bld_001"
  }
}
```

### 6.7 ExportConfig 模板

```json
{
  "export_format": "ifc",
  "final_use": "revit_view",
  "include_validation_report": true
}
```

## 7. 常见场景模板

### 7.1 场景 A：根据 CAD 图纸建模

用户输入：

```text
根据上传的 CAD 图纸建立住宅楼 BIM 模型，输出 IFC 文件，用 Revit 查看。
```

系统重点输出：

- `source_mode = cad_to_bim`
- `building_type = residential`
- `deliverables = ["ifc"]`

### 7.2 场景 B：根据 CAD 图纸建模并修改窗型

用户输入：

```text
根据上传 CAD 图纸建立 BIM 模型，将所有 800x1200 的窗替换为落地窗，导出 IFC，用 Revit 查看。
```

系统重点输出：

- `DesignIntent`
- `ElementSelector`
- `ModelPatch`

### 7.3 场景 C：根据 PDF 图纸建模并补充缺失参数

用户输入：

```text
根据上传 PDF 图纸建立办公楼 BIM 模型，首层层高 5.1m，其余层高未提供，请自动补全合理参数，导出 IFC。
```

系统重点输出：

- `MissingFields`
- `Assumptions`
- `CompletionTrace`

### 7.4 场景 D：无图纸文本建模

用户输入：

```text
项目类型：住宅楼
输入方式：无图纸，文本建模
总层数：2层
首层层高：4.5m
二层层高：3.0m
首层建筑面积：80㎡
建模要求：建立一个两层住宅楼，室内设置一个主楼梯，建筑外侧设置一个可直达二层的外楼梯；二层设置一个阳台；一层包含厨房和客厅；二层包含3个卧室，且每个卧室带独立卫生间
补充要求：未说明部分由系统按常规住宅参数自动补全
输出要求：导出 IFC 文件
最终用途：使用 Revit 打开查看结果
```

系统重点输出：

- `source_mode = text_only`
- `DesignIntent`
- `MissingFields`
- `Assumptions`
- `CompletionTrace`

说明：

- 该场景更适合方案级 BIM 建模
- 若缺少场地、朝向、开间、进深等信息，系统可以补全，但必须记录补全来源

## 8. 建议的字段优先级

当多个来源提供同一字段时，采用以下优先级：

1. 用户明确输入
2. 图纸解析结果
3. 项目表单字段
4. 模板默认值
5. AI 推断

## 9. 模板使用建议

### 9.1 对用户

- 不要求用户写技术化命令
- 只要求用户按模板说明输入关键项
- 系统负责补全缺失但非关键内容

### 9.2 对 AI 模块

- 必须强制结构化输出
- 禁止只输出自然语言说明
- 所有自动补全都必须带来源

### 9.3 对后端模块

- 只消费结构化对象
- 不直接解析用户原始自然语言
- 所有模块之间统一使用标准对象传递数据

## 10. 结论

完整建模指令模板应同时覆盖三层：

- 用户怎么说
- AI 怎么转
- 系统怎么接

对你当前项目来说，最推荐的标准形式是：

`用户自然语言模板 + AI 结构化转化模板 + 系统 JSON 数据模板`

这样可以让：

- 输入稳定
- 转化可控
- 规则清晰
- BIM 建模更容易落地
