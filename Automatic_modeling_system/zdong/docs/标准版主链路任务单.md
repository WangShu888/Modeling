# 标准版主链路任务单

本任务单用于把 [代理编排与交付方案.md](/workspace/自动建模系统/zdong/docs/代理编排与交付方案.md) 变成可执行排期。机器配置见 [agent_runbook.standard.json](/workspace/自动建模系统/zdong/app/config/agent_runbook.standard.json)。

## 1. 执行总原则

- 严格按代理顺序推进，不并行越过上游门禁。
- 每一棒都必须交结构化对象，不接受口头“差不多完成”。
- 阶段 A 的目标不是演示，而是打到正式 `model.ifc`。
- 图纸解析组必须承接现有成果继续推进，不重写已有链路。

## 2. 阶段 A：主链路打通

阶段 A 的目标是：

- 从输入走到正式导出
- 形成最小可交付版本
- 为阶段 B 的反馈闭环提供稳定交付物

### 任务 A1：接入与资产代理

代理：

- `agent_intake_asset`

输入：

- 用户上传图纸
- 用户文字需求
- 项目基础表单

输出：

- `project_id`
- `request_id`
- `asset_id[]`
- `version_id`
- `SourceBundle`

验收：

- 文本和图纸绑定到统一 `request_id`
- 资产元数据完整
- 下游可直接消费 `SourceBundle`

### 任务 A2：图纸解析代理

代理：

- `agent_drawing_parser`

前置：

- 已收到 `SourceBundle`

特别要求：

- 必须复用已有 `DWG -> DXF` 转换与解析成果
- 直接承接 [drawing_parser.py](/workspace/自动建模系统/zdong/app/drawing_parser.py)
- 直接承接 [windows_dwg_converter_service.py](/workspace/自动建模系统/zdong/app/windows_dwg_converter_service.py)

输出：

- `ParsedDrawingModel`
- `LayerMap`
- `GridMap`
- `DimensionEntities`
- `TextAnnotations`
- `StoreyCandidates`
- `PendingReview`

验收：

- `DWG` 优先转 `DXF`
- 单位、原点、朝向归一化
- 未识别图元进入 `PendingReview`

### 任务 A3：AI需求转化代理

代理：

- `agent_intent_ai`

前置：

- 已收到 `ParsedDrawingModel`

输出：

- `DesignIntent`
- `Assumptions`
- `MissingFields`
- `CompletionTrace`
- `ModelPatch`

验收：

- 输出符合 schema
- 所有补全字段可追溯
- 高风险字段显式暴露

### 任务 A4：规则与计划代理

代理：

- `agent_rules_planner`

前置：

- 已收到 `DesignIntent`

输出：

- `RuleCheckResult`
- `RuleViolations`
- `RuleWarnings`
- `ModelingPlan`
- `AffectedModules`
- `RegenerationScope`

验收：

- 先规则后计划
- 规则结果可解释
- 计划可回放

### 任务 A5：BIM执行代理

代理：

- `agent_bim_executor`

前置：

- 已收到 `ModelingPlan`

输出：

- 内部 `BIM` 语义模型
- `IfcProject / IfcSite / IfcBuilding / IfcBuildingStorey`
- 构件索引与版本映射

验收：

- 楼层与宿主关系完整
- `GUID` 稳定
- 模型可被自检读取

### 任务 A6：质量回退与交付代理

代理：

- `agent_validation_delivery`

前置：

- 已收到内部 `BIM` 语义模型

输出：

- `ValidationReport`
- `RollbackTarget`
- `RepairInstruction`
- `model.ifc`
- `validation.json`
- `intent.json`
- `ExportBundle`

验收：

- `fatal/error` 阻断导出
- 失败时精确回退
- 通过后正式导出

## 3. 阶段 B：交付闭环补齐

### 任务 B1：工作台与反馈代理

代理：

- `agent_workbench_feedback`

前置：

- 已收到 `ExportBundle`

输出：

- 版本查看界面
- 导出结果下载
- `FeedbackRequest`

验收：

- 用户可查看交付版本
- 用户可下载交付产物
- 用户反馈可重新进入主链路

## 4. 阶段门禁

阶段 A 放行条件：

1. `SourceBundle` 已形成
2. `ParsedDrawingModel` 已形成
3. `DesignIntent` 已形成
4. `ModelingPlan` 已形成
5. 内部 `BIM` 语义模型已形成
6. `ValidationReport` 已通过并导出 `model.ifc`

阶段 B 放行条件：

1. `ExportBundle` 已可展示
2. 下载能力已接通
3. 反馈可转为新的请求

## 5. 建议排期

如果按一个稳定版本来推，建议顺序是：

1. 先完成 A1 到 A4，保证“输入到计划”稳定
2. 再完成 A5 和 A6，拿到首个正式导出版本
3. 最后完成 B1，把交付闭环补齐

## 6. 执行提醒

- 阶段 A 没完成前，不要把精力分散到过多前端体验优化。
- 图纸解析代理要优先接现有能力，不要为重构耽误主链路。
- 质量回退与交付代理必须是正式放行门，不只是日志记录器。
