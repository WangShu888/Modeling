# 构件识别专项检测报告

日期：`2026-04-02`

## 1. 结论

- 问题已确认存在。
- 当前“构件识别不够准确”并非单一误分类问题，更明显的是识别结果存在截断风险。

## 2. 检测结果

- 真实 DXF 解析后 `detected_entities = 200`
- 该值等于当前配置上限 `_MAX_DETECTED_ENTITIES = 200`
- 系统已新增专项告警：
  - `entity_detection_truncated`

## 3. 根因判断

- 解析器对候选构件的收集存在硬上限。
- 一旦复杂图纸达到上限，后续候选会被静默丢弃。
- 因此当前输出更接近“截断样本”，而不是“完整识别结果”。

## 4. 影响

- 墙、门、窗等构件总量可能被低估。
- 下游 BIM 构件数量和空间完整性会随之偏差。
- 报告中若只看最终 `IfcWall / IfcDoor / IfcWindow` 数量，容易误判为识别已完成。

## 5. 本次补充检测

- 解析器现在会在达到识别上限时输出 `entity_detection_truncated`
- 规则层会继续向 `rule_check` 透传该风险
- 自动化测试已覆盖该告警

## 6. 后续建议

1. 把识别上限命中次数和被丢弃候选数单独统计出来。
2. 增加按图层、按构件类型的识别覆盖率分析，而不是只看总量。
3. 如需提高真实准确率，应优先处理图层映射和上限截断，再处理细粒度分类。
