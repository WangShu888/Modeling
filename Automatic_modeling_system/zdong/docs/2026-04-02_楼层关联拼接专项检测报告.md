# 楼层关联拼接专项检测报告

日期：`2026-04-02`

## 1. 结论

- 问题已确认存在。
- 当前系统能识别到多楼层标题，但不能把同一资产内的多楼层几何切开并建立关联拼接。

## 2. 检测结果

- 真实样本显式楼层候选包含 `1F / 2F / RF`
- 最终 `source_storey_keys` 只有 `['1F']`
- 当前系统已新增专项告警：
  - `multi_storey_asset_collapsed`

## 3. 根因判断

- `infer_parsed_asset_storeys()` 仍以 `asset_name -> 单个 storey_key` 的方式输出结果。
- `BimEngine._source_layouts()` 也是按资产级归组几何，而不是按单个楼层视图分段。
- 因此只要一张 DXF 同时包含多个楼层平面，当前实现就会把它们压缩到同一个楼层键下。

## 4. 影响

- 楼层无法可靠拆分。
- 楼层之间无法自动建立拼接关系。
- 后续楼板、墙体、门窗的楼层归属会被整体折叠，影响最终 IFC 结构。

## 5. 本次补充检测

- 解析器现在会在发现“单资产含多个显式楼层候选且存在几何实体”时输出 `multi_storey_asset_collapsed`
- 规则层会继续透传该风险
- 自动化测试已覆盖该告警

## 6. 后续建议

1. 在解析层实现“视图标题 + 版面边框 + 几何聚类”的联合切分，先把单资产拆成楼层视图片段。
2. 在楼层推断层改为 `asset fragment -> storey_key`，不要继续使用 `asset -> storey_key` 单值模型。
3. 在 BIM 建模层为跨楼层复用、镜像、典型层继承建立显式关联元数据，而不是仅复制最后一个楼层布局。
