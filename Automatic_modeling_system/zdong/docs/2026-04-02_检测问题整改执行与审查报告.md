## 3. 执行修改概述

### 3.1 A+B 线：图纸解析与楼层片段

涉及文件：

- `app/models.py`
- `app/drawing_parser.py`
- `app/storey_inference.py`
- `tests/test_drawing_parser_view_markers.py`
- `tests/test_storey_inference.py`

已完成修改：

1. `ParsedDrawingModel` 新增片段级与统计字段：
   - `detected_entities_total`
   - `detected_entities_emitted`
   - `detected_entities_dropped`
   - `detected_entities_source_summary`
   - `fragments`
2. 新增 `DrawingFragmentRecord`，用于承载：
   - `fragment_id`
   - `fragment_title`
   - `fragment_role`
   - `storey_key`
   - `bbox`
3. 取消 `detected_entities` 的硬截断返回，改为全量输出并显式统计 dropped。
4. 通过 `fragment_id`、`fragment_storey_key`、`fragment_role` 将解析实体回填到楼层片段事实。
5. 新增 `infer_parsed_fragment_storeys()`，将楼层推断从 `asset -> storey` 提升为 `fragment -> storey`。
6. 将楼层候选文本的 `bbox` 与 `source_ref` 写入 `StoreyCandidateRecord`，使真实 DXF 的楼层标题能够直接成为片段定位锚点。
7. 将实体分配到片段的回退规则从“按 y 最近”修正为“按二维距离最近”，避免 `1F/2F` 标题位于同一水平带时全部落入 `1F`。
8. 收敛 `multi_storey_asset_collapsed` 诊断逻辑：只有在显式多楼层存在、但实体仍集中在过少楼层时才报警，避免已完成分层后继续误报。

### 3.2 C+D 线：BIM 落模、门禁与导出

涉及文件：

- `app/planning.py`
- `app/pipeline.py`
- `tests/test_bim_engine_source_geometry.py`
- `tests/test_validation_export.py`

已完成修改：

1. `ConfigurableRuleEngine` 将以下问题升级为正式阻断：
   - `drawing.review.entity_detection_truncated`
   - `drawing.review.multi_storey_asset_collapsed`
2. `BimEngine` 开始显式消费片段来源，补充：
   - `source_storey_key`
   - `source_fragment_id`
   - `source_fragment_ids`
   - `storey_layout_trace`
   - `count_reconciliation`
3. `ValidationService` 补充正式门禁与校验：
   - 正式阻断项提升
   - 构件数量对账
   - 位置边界校验
   - `gate_trace` 扩展
4. `ExportService` 读取 `formal_blocking_issues`，即使 `validation.status != failed` 也会阻断正式导出。
5. 修正门窗边界校验：对 `door_block` / `window_block` 采用中心点判定，避免靠边门窗被误判越界。

### 3.3 主线程集成修正

主线程额外完成了两项关键收口：

1. 修正 `pipeline.py` 对解析元数据的消费口径，使其兼容 `fragment_storey_key` 与 `fragment_role`，避免解析已分层但 BIM 侧再次按资产级折叠。
2. 修正 `drawing_parser.py` 中 fragment 标题 bbox 匹配的重复扫描问题，将其从反复遍历文本改为索引查找。
